# API

Base URL for local development:

```text
http://localhost:8000
```

## GET /health

Purpose: check backend availability.

Response:

```json
{
  "status": "ok",
  "app": "SmartBuyAgent",
  "version": "0.1.0"
}
```

## POST /api/chat

Purpose: run a non-streaming shopping-guide or knowledge-answer request.

Request:

```json
{
  "query": "预算3000，推荐一款拍照好的手机",
  "session_id": null,
  "debug": true
}
```

Response:

```json
{
  "answer": "...",
  "product_cards": [
    {
      "product_id": "phone_001",
      "title": "...",
      "brand": "...",
      "price": 2999,
      "image_url": "https://example.com/images/phone_001.jpg",
      "tags": ["拍照好", "性价比"],
      "attributes": {
        "存储容量": "256GB"
      },
      "source_url": "https://example.com/products/phone_001",
      "compare_url": "https://example.com/compare/phone_001",
      "recommend_reason": "..."
    }
  ],
  "citations": [
    {
      "chunk_id": "doc_phone_phone_camera_guide_chunk_001",
      "title": "手机拍照选购指南",
      "section": "...",
      "section_path": "...",
      "source_file": "data/knowledge_docs/phone/phone_camera_guide.md",
      "content_preview": "...",
      "score": 0.82
    }
  ],
  "trace": [
    {
      "step": "query_understanding",
      "intent": "shopping_guide"
    }
  ],
  "session_id": "generated-or-reused-session-id"
}
```

Notes:

- `query` is required and cannot be blank.
- `session_id` is optional. If omitted, the backend creates a new session ID.
- `debug=false` returns an empty `trace` list.
- Product cards and citations are not generated by the LLM.

## POST /api/chat/stream

Purpose: run the same chat workflow and return Server-Sent Events for debug display.

Request:

```json
{
  "query": "预算3000，推荐一款拍照好的手机",
  "session_id": null,
  "debug": true
}
```

Response content type:

```text
text/event-stream
```

Events:

```text
event: session
data: {"session_id":"..."}

event: trace
data: {"step":"query_understanding","intent":"shopping_guide"}

event: result
data: {"answer":"...","product_cards":[],"citations":[],"trace":[],"session_id":"..."}

event: done
data: {"status":"ok"}
```

On stream-level failure:

```text
event: error
data: {"message":"chat stream failed"}

event: done
data: {"status":"error"}
```

Notes:

- Stage 9.1 streaming emits trace after the AgentWorkflow completes.
- This is trace streaming, not token-level LLM streaming.
- Memory saving is handled by the API layer and stores the original query.

## POST /api/feedback

Purpose: store user feedback for an answer.

Request:

```json
{
  "session_id": "session-id",
  "turn_id": null,
  "rating": "helpful",
  "reason": "recommendation_relevant",
  "comment": "The answer was useful.",
  "query": "预算3000，推荐一款拍照好的手机",
  "answer_preview": "Short answer preview..."
}
```

Response:

```json
{
  "id": 1,
  "status": "saved"
}
```

Notes:

- `rating` must be `helpful` or `not_helpful`.
- `comment` is limited to 1000 characters.
- `answer_preview` is truncated server-side to 500 characters.
- Feedback is stored for evaluation and does not affect current ranking.
